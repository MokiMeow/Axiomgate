# Project Naming

## Working name

# AxiomGate

**Tagline:** Plan. Govern. Execute. Prove.

The name reflects two core ideas:

- **Axiom:** the mission contract and policies that define what must remain true.
- **Gate:** consequential actions and completion claims must pass explicit checks.

## Product descriptor

Use this descriptor in README, Devpost, and demo materials:

> A local-first governed runtime for verifiable coding-agent missions.

## Naming rules

- Treat AxiomGate as a working product name until a final trademark, package, repository, and domain review is completed.
- Do not rename internal modules casually during implementation.
- Module names should describe responsibility rather than marketing:
  - Mission Compiler
  - Runway
  - Environment Guard
  - Codex Runtime
  - Verification Engine
  - Evidence Gate
- PatchPilot remains the name of the existing verification technology where applicable.
